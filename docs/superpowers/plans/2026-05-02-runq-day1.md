# RunQ Day 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first runnable RunQ open source skeleton with protocol docs, event validation, local SQLite storage, and sample ingestion.

**Architecture:** Use dependency-free Node.js ESM modules so the project can run immediately. The core package exposes event validation, a local SQLite store backed by `node:sqlite`, and a CLI for ingesting sample RunQ events.

**Tech Stack:** Node.js 22+, built-in `node:test`, built-in experimental `node:sqlite`, JavaScript ESM with JSDoc-style structure.

---

## Files

- Create `package.json`: project metadata and scripts.
- Create `README.md`: public-facing RunQ introduction.
- Create `protocol/protocol-v0.md`: protocol draft.
- Create `protocol/events.schema.json`: event envelope JSON schema.
- Create `src/schema.js`: event schema constants and validation.
- Create `src/store.js`: SQLite-backed local event store.
- Create `src/cli.js`: CLI for ingesting JSON events and listing sessions.
- Create `examples/sessions/basic-run.json`: sample event stream.
- Create `test/schema.test.js`: event validation tests.
- Create `test/store.test.js`: SQLite store tests.
- Create `test/cli.test.js`: CLI behavior tests.

# Code of Conduct

RunQ is an open source project for developers working on coding-agent reliability. We want technical disagreement to be direct, specific, and useful without making participation hostile.

## Expected Behavior

- Be respectful of different experience levels, backgrounds, and viewpoints.
- Keep criticism focused on ideas, code, data, and tradeoffs.
- Assume good intent when clarification is reasonable.
- Accept maintainer decisions once a discussion has been resolved.
- Report security issues privately through the process in `SECURITY.md`.

## Unacceptable Behavior

- Harassment, intimidation, threats, or sustained personal attacks.
- Discriminatory language or behavior.
- Publishing private information without explicit permission.
- Trolling, repeated off-topic disruption, or bad-faith demands.
- Pressuring maintainers to disclose private security reports before fixes are ready.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; temporarily limit participation; or permanently ban contributors who violate this code.

If you need to report a conduct issue, open a private maintainer contact channel through the project owner. Include links, screenshots, and relevant context. Maintainers will review reports privately and respond with the least disruptive action that protects the project.
